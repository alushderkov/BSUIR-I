# Сервер управления задачами на GraphQL

## Теоретическая часть

### Что такое GraphQL?

GraphQL - это язык запросов для API и среда выполнения для выполнения этих запросов с существующими данными. В отличие от REST API, GraphQL предоставляет клиентам возможность запрашивать именно те данные, которые им нужны, не больше и не меньше.

#### Основные преимущества GraphQL:

1. **Точность данных**: Клиенты получают только те данные, которые они запросили
2. **Единая конечная точка**: Все запросы отправляются на один URL
3. **Строгая типизация**: Схема GraphQL определяет типы данных и их отношения
4. **Интроспекция**: API самоописываемое, что упрощает разработку

### Структура GraphQL API

#### 1. Схема (Schema)
Схема определяет типы данных и операции, которые можно выполнять с этими данными. В нашем случае схема включает:

- **Типы данных**:
  - `User`: Пользователь системы
  - `Task`: Задача
  - `TaskComplexity`: Перечисление сложности задачи (EASY, MEDIUM, HARD)
  - `AuthPayload`: Данные аутентификации

- **Запросы (Queries)**:
  - Получение списка задач
  - Получение задачи по ID
  - Получение информации о текущем пользователе
  - Фильтрация задач по сложности
  - Фильтрация задач по тегам
  - Фильтрация задач по диапазону дат

- **Мутации (Mutations)**:
  - Создание задачи
  - Обновление задачи
  - Удаление задачи
  - Регистрация пользователя
  - Вход в систему

#### 2. Резолверы (Resolvers)
Резолверы - это функции, которые определяют, как получить данные для каждого поля в схеме. Они обрабатывают:
- Запросы данных
- Мутации данных
- Аутентификацию
- Валидацию

Пример реализации резолвера для создания задачи:

```javascript
const resolvers = {
  Mutation: {
    createTask: async (_, { title, description, userId, date, complexity, tags, deadline }, context) => {
      // Проверка авторизации
      const contextUserId = getUserId(context);
      if (contextUserId !== userId) {
        throw new Error('Not authorized');
      }

      // Чтение текущих данных
      const data = readData(tasksPath);

      // Создание новой задачи
      const newTask = {
        id: Date.now().toString(),
        title,
        description,
        completed: false,
        createdAt: new Date().toISOString(),
        userId,
        date,
        complexity,
        tags,
        deadline
      };

      // Сохранение задачи
      data.tasks.push(newTask);
      writeData(tasksPath, data);

      return newTask;
    }
  }
};
```

В этом примере резолвер:
1. Принимает параметры из GraphQL запроса
2. Проверяет авторизацию пользователя
3. Создает новую задачу с уникальным ID
4. Сохраняет задачу в хранилище
5. Возвращает созданную задачу

Каждый резолвер имеет доступ к:
- `_`: Родительский объект (не используется в корневых резолверах)
- `args`: Аргументы, переданные в запрос
- `context`: Контекст запроса (содержит информацию о пользователе, заголовки и т.д.)
- `info`: Информация о самом запросе

## Реализация

### Технологический стек

- **Node.js**: Среда выполнения JavaScript
- **Express**: Веб-фреймворк
- **Apollo Server**: Реализация GraphQL сервера
- **CORS**: Middleware для обработки кросс-доменных запросов

### Структура проекта

```
server/
├── src/
│   ├── data/           # JSON файлы для хранения данных
│   ├── schema/         # GraphQL схема
│   ├── resolvers/      # Резолверы
│   ├── utils/          # Вспомогательные функции
│   └── index.js        # Точка входа
├── package.json
└── README.md
```

### Основные компоненты

#### 1. Сервер (index.js)
- Настройка Express приложения
- Интеграция Apollo Server
- Настройка CORS
- Запуск сервера на порту 4000

#### 2. Схема (typeDefs.js)
Определяет структуру данных и доступные операции:

```graphql
type Task {
  id: ID!
  title: String!
  description: String
  completed: Boolean!
  createdAt: String!
  userId: ID!
  date: String!
  complexity: TaskComplexity!
  tags: [String!]!
  deadline: String
}
```

#### 3. Резолверы (resolvers/index.js)
Реализует логику обработки запросов:

- **Query резолверы**:
  - `tasks`: Получение всех задач пользователя
  - `task`: Получение задачи по ID
  - `tasksByComplexity`: Фильтрация по сложности
  - `tasksByTag`: Фильтрация по тегам
  - `tasksByDateRange`: Фильтрация по датам

- **Mutation резолверы**:
  - `createTask`: Создание новой задачи
  - `updateTask`: Обновление существующей задачи
  - `deleteTask`: Удаление задачи
  - `register`: Регистрация нового пользователя
  - `login`: Аутентификация пользователя

### Безопасность

1. **Аутентификация**:
   - JWT токены для авторизации
   - Хеширование паролей
   - Проверка прав доступа к задачам

2. **Валидация данных**:
   - Проверка обязательных полей
   - Валидация типов данных
   - Проверка уникальности пользователей

### Хранение данных

В текущей реализации данные хранятся в JSON файлах:
- `tasks.json`: Хранение задач
- `users.json`: Хранение пользователей

### Запуск проекта

1. Установка зависимостей:
```bash
npm install
```

2. Запуск в режиме разработки:
```bash
npm run dev
```

3. Запуск в production режиме:
```bash
npm start
```

Сервер будет доступен по адресу: `http://localhost:4000/graphql`

### Примеры запросов

#### Создание задачи:
```graphql
mutation {
  createTask(
    title: "Новая задача"
    description: "Описание задачи"
    userId: "user_id"
    date: "2024-03-20"
    complexity: MEDIUM
    tags: ["важное", "срочное"]
    deadline: "2024-03-25"
  ) {
    id
    title
    completed
  }
}
```

#### Получение задач:
```graphql
query {
  tasks {
    id
    title
    description
    completed
    complexity
    tags
    deadline
  }
}
```
