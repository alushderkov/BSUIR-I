### Часть 1: Что такое коды G и L?

Ваша задача — построить «переводчик», который принимает число на одном языке (Код G) и выдает его на другом языке (Код L).

#### 1. Код G = 7421 (Входные данные)

Это **взвешенный код**. Это значит, что каждый бит (разряд) имеет свой "вес". В обычном двоичном коде веса — это 8, 4, 2, 1. А здесь веса — **7, 4, 2, 1**.

Чтобы понять, какое число зашифровано, мы просто складываем веса тех битов, где стоит единичка.
_Переменные:_ Обозначим входные биты как $G_3, G_2, G_1, G_0$.

- **0:** `0000` (сумма 0)
- **1:** `0001` (вес 1)
- **2:** `0010` (вес 2)
- **3:** `0011` (2 + 1)
- **4:** `0100` (вес 4)
- **5:** `0101` (4 + 1)
- **6:** `0110` (4 + 2)
- **7:** `1000` (вес 7) — _Обратите внимание: теоретически 7 можно записать и как 0111 (4+2+1), но в вашем варианте используется 1000, это стандарт для этого кода._
- **8:** `1001` (7 + 1)
- **9:** `1010` (7 + 2)

#### 2. Код L = С избытком 3 (Excess-3) (Выходные данные)

Это **невзвешенный код**. Он получается очень просто: берется обычное число, к нему прибавляется 3, и результат записывается в обычном двоичном коде (8421).
_Переменные:_ Обозначим выходные биты как $L_3, L_2, L_1, L_0$.

- **0:** $0 + 3 = 3$ $\rightarrow$ `0011`
- **1:** $1 + 3 = 4$ $\rightarrow$ `0100`
- **...**
- **9:** $9 + 3 = 12$ $\rightarrow$ `1100`

### 1. Что такое «Взвешенный код»? (Пример с монетами)

Представьте, что каждый выключатель (бит) — это кошелек, в котором лежит монета определенного номинала (**Веса**).

Если вы включаете выключатель (ставите **1**), вы "берете" эту монету. Если выключаете (ставите **0**), монета остается лежать, вы её не считаете.

#### Обычный код (8421)

В классической информатике монеты имеют достоинство: **8, 4, 2, 1**.

- Чтобы получить число **5**, вы берете монеты **4** и **1**.
  - Код: `0 1 0 1` (8 не берем, **4 берем**, 2 не берем, **1 берем**).
  - Сумма: 4 + 1 = 5.

#### Ваш код G (7421) — Вариант 24

В вашем задании правила игры изменились. У вас в кошельках лежат монеты другого достоинства:
**7, 4, 2, 1**.

Давайте посмотрим, как с помощью этих странных монет набрать обычные цифры.

- **Цифра 0:** Не берем ничего.
  - Код: `0000`

- **Цифра 1:** Нам нужна сумма 1. Берем только последнюю монету (1).
  - Код: `0001` (0+0+0+**1**)

- **Цифра 2:** Берем монету (2).
  - Код: `0010` (0+0+**2**+0)

- **Цифра 3:** Как набрать 3? Нужно взять монету (2) и монету (1).
  - Код: `0011` (0+0+**2**+**1** = 3)

- **Цифра 4:** Берем монету (4).
  - Код: `0100` (0+**4**+0+0 = 4)

- **Цифра 5:** Нам нужно 5. Это (4) + (1).
  - Код: `0101` (0+**4**+0+**1** = 5)

- **Цифра 6:** Это (4) + (2).
  - Код: `0110` (0+**4**+**2**+0 = 6)

- **Цифра 7:** Вот тут интересно! У нас есть целая монета (7). Мы просто берем её одну.
  - Код: `1000` (**7**+0+0+0 = 7)
  - _(Можно было бы набрать 4+2+1, но в этом коде принято использовать старшую монету, если она влезает)._

- **Цифра 9:** Это (7) + (2).
  - Код: `1010` (**7**+0+**2**+0 = 9)

**Итог:** Ваш код **G = 7421** — это просто инструкция, какие "монеты" (биты) нужно сложить, чтобы получить задуманное число.

### 2. Что такое код L (С избытком 3)?

Здесь логика другая. Это уже не монеты. Это просто математическое правило для шифрования.

**Правило:** "Возьми обычное число, прибавь к нему 3, и результат запиши обычными двоичными монетами (8-4-2-1)".

Пример:

- Мы хотим закодировать цифру **5**.
- Действие: $5 + 3 = 8$.
- Как записать **8** обычными битами (8-4-2-1)? Нужно включить первый выключатель (который 8).
- Результат: `1000`.

Пример 2:

- Мы хотим закодировать цифру **0**.
- Действие: $0 + 3 = 3$.
- Как записать **3**? Это 2 + 1.
- Результат: `0011`.

### Часть 3: Задание 7 (Структурный метод)

**Суть:** Здесь мы должны построить схему "руками" из логических элементов (вентилей), как инженер паяет микросхему.
В задании сказано использовать мультиплексоры (MUX2) и инверторы (NOT).

#### Как происходил расчет (Минимизация)?

Инженер берет таблицу истинности (из Части 1) и для каждого бита выхода ($L_3, L_2, L_1, L_0$) составляет уравнение, чтобы понять, при каких $G$ этот бит равен 1.

_Примечание:_ В предоставленном вами коде (Structural) логика **содержит ошибку или взята из другого варианта**, особенно для бита $L_0$.

- В коде написано: `out_value(0) <= '1';` (Бит L0 всегда горит).
- В таблице: для цифры 1 выход `0100` (последний бит 0).
- **Вывод:** Если вы сдадите этот код, преподаватель может заметить, что схема выдает неверные значения. Но давайте разберем _структуру_ VHDL, так как задание требует описать именно этот код.

#### Разбор кода VHDL (Структурный):

1.  **Компоненты (`component`):**
    Мы объявляем "детали", которые будем использовать:
    - `NOT_1`: Инвертор (превращает 0 в 1 и наоборот).
    - `AND_N_MUX` и `OR_N_MUX`: Это хитрые блоки.
      - Обычно логическое **И** ($A \cdot B$) и **ИЛИ** ($A+B$) делают на базовых вентилях. Но в задании просят MUX (мультиплексоры).
      - Мультиплексор — это управляемый переключатель. Блоки `AND_N_MUX` внутри себя построены из мультиплексоров, которые настроены работать как логическое И/ИЛИ.

2.  **Сигналы (`signal`):**
    Это внутренние провода.
    - `not_g3`...: Это инвертированные входные сигналы ( $\overline{G_3}$ ).
    - `l3_terms`: Промежуточные результаты вычислений.

3.  **Реализация уравнений (Port Map):**
    Код пытается реализовать формулы. Посмотрим на пример для $L_3$:
    - Формула в комментарии: $L_3 = G_3 \cdot \overline{G_2} + G_2 \cdot G_1$
    - **Строка `L3_TERM1_AND`:**
      - Берет `in_value(3)` ($G_3$) и `not_g2` ($\overline{G_2}$).
      - Выполняет операцию И. Результат: `l3_term1`.
    - **Строка `L3_OR`:**
      - Собирает результаты ("термы") и объединяет их через ИЛИ.
      - Результат идет на выход `out_value(3)`.

#### Что такое MUX (Мультиплексор) и причем тут логика?

В задании сказано "использовать только mux2".
Мультиплексор `mux2(Input0, Input1, Select)` работает так:

- Если Select = 0, на выход идет Input0.
- Если Select = 1, на выход идет Input1.

Как сделать логическое **И** ($A \cdot B$) из MUX?

- Подключим так: `MUX(0, A, B)`.
- Если B=0, выход 0. Если B=1, выход равен A. Это и есть $A \cdot B$.
  В вашем коде компоненты `AND_N_MUX`, скорее всего, скрывают внутри именно такую конструкцию.

### 2. Формулы для каждого выхода

Обозначения: Входные биты — $G_3, G_2, G_1, G_0$.

#### **Выход L3 (Самый старший бит)**

Он равен **1**, когда число $\ge 5$ (т.е. 5, 6, 7, 8, 9).

- **Формула:**
  $$L_3 = G_3 + G_2 \cdot (G_1 + G_0)$$
  _Смысл:_ Бит горит, если входное число 7 или больше ($G_3=1$), **ИЛИ** если есть четверка ($G_2=1$) вместе с двойкой или единицей.

#### **Выход L2**

Он равен **1** для чисел 1, 2, 3, 4 и 9.

- **Формула:**
  $$L_2 = G_3 \cdot G_1 + \overline{G_3} \cdot (G_2 \oplus (G_1 + G_0))$$
  _Смысл:_ Довольно сложная логика. Если $G_3$ нет, то $L_2$ ведет себя как исключающее ИЛИ между $G_2$ и суммой младших битов.

#### **Выход L1**

Он равен **1** для чисел 0, 3, 4, 7, 8.

- **Формула:**

  _Смысл:_ $L_1$ горит, если $G_1$ и $G_0$ одинаковые (оба 0 или оба 1), либо если есть специфическая комбинация для восьмерки.

#### **Выход L0 (Младший бит)**

Он равен **1** для чисел 0, 2, 4, 6, 8.

- **Формула:**
  $$L_0 = G_3 \odot G_0$$
  _(Или: $L_0 = G_3 \cdot G_0 + \overline{G_3} \cdot \overline{G_0}$)_

### 3. Как это превратить в VHDL (Структурный, только MUX и NOT)?

Вам нужно собрать эти формулы, используя компонент `MUX2`.
Мультиплексор `MUX2(input0, input1, select)` работает так: если `select=0`, берет `input0`, иначе `input1`.

Вот шпаргалка, как заменить логику на MUX:

1.  **AND (И):** $A \cdot B$ $\rightarrow$ `MUX('0', B, A)` (Если A=0, то 0, если A=1, то B).
2.  **OR (ИЛИ):** $A + B$ $\rightarrow$ `MUX(B, '1', A)` (Если A=0, то B, если A=1, то 1).
3.  **XNOR (Равно):** $A \odot B$ $\rightarrow$ `MUX(NOT B, B, A)`.

**Пример исправления части кода для L0 (самое простое):**

```vhdl
-- L0 = G3 XNOR G0
-- На языке MUX: Если G3=0, то (NOT G0), если G3=1, то G0.

-- Сначала инвертируем G0
NOT_G0_INST: NOT_1 port map (a => in_value(0), q => not_g0);

-- MUX для L0
MUX_L0: MUX2
    port map (
        d0 => not_g0,      -- Когда select (G3) = 0
        d1 => in_value(0), -- Когда select (G3) = 1
        s  => in_value(3), -- G3
        y  => out_value(0)
    );
```

Если вы будете сдавать работу, используйте эти формулы. Они точно переводят 7421 в Excess-3.

Понял вашу задачу. Вы абсолютно правы: для структурного описания в базисе «И/ИЛИ/НЕ» (которое мы потом переведем на MUX) нужно использовать **ДНФ (Дизъюнктивную Нормальную Форму)**.

Это значит, мы не используем XOR/XNOR, а представляем каждый выход $L$ как **сумму произведений** (Sum of Products).

Ниже приведен полный расчет формул через минимизацию (Карты Карно) и готовый VHDL код, где логика **И** и **ИЛИ** реализована на мультиплексорах `mux2`.

### 1. Расчет формул (Минимизация ДНФ)

У нас есть вход $G_3, G_2, G_1, G_0$ и выход $L_3, L_2, L_1, L_0$.
Неиспользуемые комбинации (1011...1111) считаем _don't care_ (можно брать 0 или 1 для упрощения).

#### **Выход L3**

Должен быть **1** для цифр 5, 6, 7, 8, 9.

- Минтермы:
  - $G_3$ (покрывает 7, 8, 9)
  - $G_2 \cdot G_1$ (покрывает 6)
  - $G_2 \cdot G_0$ (покрывает 5)
- **Итоговая формула:**
  $$L_3 = G_3 + G_2 \cdot G_1 + G_2 \cdot G_0$$

#### **Выход L2**

Должен быть **1** для цифр 1, 2, 3, 4, 9.

- Минтермы:
  - $G_3 \cdot G_1$ (для цифры 9: `1010`. Бит $G_1$ отличает 9 от 7 и 8).
  - $\overline{G_2} \cdot G_1$ (для 2 и 3).
  - $\overline{G_2} \cdot G_0$ (для 1 и 3).
  - $G_2 \cdot \overline{G_1} \cdot \overline{G_0}$ (для цифры 4: `0100`).
- **Итоговая формула:**
  $$L_2 = G_3 G_1 + \overline{G_2} G_1 + \overline{G_2} G_0 + G_2 \overline{G_1} \overline{G_0}$$

#### **Выход L1**

Должен быть **1** для цифр 0, 3, 4, 7, 8.

- Минтермы:
  - $G_3 \cdot \overline{G_1}$ (для 7 и 8).
  - $\overline{G_1} \cdot \overline{G_0}$ (для 0 и 4).
  - $\overline{G_3} \cdot G_1 \cdot G_0$ (для 3).
- **Итоговая формула:**
  $$L_1 = G_3 \overline{G_1} + \overline{G_1} \overline{G_0} + \overline{G_3} G_1 G_0$$

#### **Выход L0**

Должен быть **1** для цифр 0, 2, 4, 6, 8.

- Здесь нам нужно описать "равнозначность" через ДНФ.
- Минтермы:
  - $\overline{G_3} \cdot \overline{G_0}$ (для 0, 2, 4, 6 — у них всех $G_3=0$ и $G_0=0$).
  - $G_3 \cdot G_0$ (для 8 — `1001`, $G_3=1, G_0=1$).
- **Итоговая формула:**
  $$L_0 = \overline{G_3} \overline{G_0} + G_3 G_0$$

---

### 2. Как реализовать это на MUX2

Мультиплексор `MUX2(D0, D1, S)` работает так: `IF S=0 THEN D0 ELSE D1`.

1.  **Логическое И ($A \cdot B$):**
    `MUX(0, B, A)` $\rightarrow$ Если A=0, то 0. Если A=1, то B.
2.  **Логическое ИЛИ ($A + B$):**
    `MUX(B, 1, A)` $\rightarrow$ Если A=0, то B. Если A=1, то 1.
3.  **Инвертор (NOT A):**
    У нас есть отдельный вентиль `INV` по условию.

---

### 3. VHDL Код (Структурный)

В этом коде я последовательно собираю эти формулы, соединяя MUX-ы проводами (signals).

```vhdl
library IEEE;
use IEEE.STD_LOGIC_1164.ALL;

entity CODE_TRANSLATOR_STRUCT is
    port (
        sw_in   : in  std_logic_vector(3 downto 0); -- Вход G (G3..G0)
        led_out : out std_logic_vector(3 downto 0)  -- Выход L (L3..L0)
    );
end CODE_TRANSLATOR_STRUCT;

architecture structural of CODE_TRANSLATOR_STRUCT is

    -- 1. Компонент Мультиплексор 2-в-1
    component MUX2 is
        port (
            d0 : in  std_logic;
            d1 : in  std_logic;
            s  : in  std_logic;
            y  : out std_logic
        );
    end component;

    -- 2. Компонент Инвертор
    component INV is
        port (
            a : in  std_logic;
            y : out std_logic
        );
    end component;

    -- Сигналы для входных битов и их инверсий
    signal g3, g2, g1, g0       : std_logic;
    signal ng3, ng2, ng1, ng0   : std_logic;

    -- Промежуточные сигналы (термы произведений и суммы)
    -- Для L3
    signal l3_term_g2g1, l3_term_g2g0 : std_logic;
    signal l3_sum1 : std_logic;

    -- Для L2
    signal l2_term1, l2_term2, l2_term3, l2_term4 : std_logic;
    signal l2_sum1, l2_sum2, l2_sum3 : std_logic;

    -- Для L1
    signal l1_term1, l1_term2, l1_term3 : std_logic;
    signal l1_term3_part : std_logic; -- Промежуточный для 3-входового И
    signal l1_sum1, l1_sum2 : std_logic;

    -- Для L0
    signal l0_term1, l0_term2 : std_logic;

begin
    -- Разводка входов
    g3 <= sw_in(3);
    g2 <= sw_in(2);
    g1 <= sw_in(1);
    g0 <= sw_in(0);

    -- Инверторы (создаем НЕ-сигналы)
    INV_3: INV port map (a => g3, y => ng3);
    INV_2: INV port map (a => g2, y => ng2);
    INV_1: INV port map (a => g1, y => ng1);
    INV_0: INV port map (a => g0, y => ng0);

    ------------------------------------------------------------------------
    -- РЕАЛИЗАЦИЯ L3 = G3 + (G2*G1) + (G2*G0)
    ------------------------------------------------------------------------

    -- Term: G2 * G1 -> MUX(0, G1, G2)
    AND_L3_1: MUX2 port map (d0 => '0', d1 => g1, s => g2, y => l3_term_g2g1);

    -- Term: G2 * G0 -> MUX(0, G0, G2)
    AND_L3_2: MUX2 port map (d0 => '0', d1 => g0, s => g2, y => l3_term_g2g0);

    -- OR Sum: G3 + Term1 -> MUX(Term1, 1, G3)
    OR_L3_1:  MUX2 port map (d0 => l3_term_g2g1, d1 => '1', s => g3, y => l3_sum1);

    -- OR Final: Sum1 + Term2
    OR_L3_2:  MUX2 port map (d0 => l3_term_g2g0, d1 => '1', s => l3_sum1, y => led_out(3));

    ------------------------------------------------------------------------
    -- РЕАЛИЗАЦИЯ L2 = (G3*G1) + (nG2*G1) + (nG2*G0) + (G2*nG1*nG0)
    ------------------------------------------------------------------------

    -- Term1: G3 * G1
    AND_L2_1: MUX2 port map (d0 => '0', d1 => g1, s => g3, y => l2_term1);

    -- Term2: nG2 * G1
    AND_L2_2: MUX2 port map (d0 => '0', d1 => g1, s => ng2, y => l2_term2);

    -- Term3: nG2 * G0
    AND_L2_3: MUX2 port map (d0 => '0', d1 => g0, s => ng2, y => l2_term3);

    -- Term4: G2 * nG1 * nG0 (Тройное И)
    -- Сначала (G2 * nG1) -> tmp, потом (tmp * nG0) -> res
    signal l2_term4_tmp : std_logic;
    AND_L2_4a: MUX2 port map (d0 => '0', d1 => ng1, s => g2, y => l2_term4_tmp);
    AND_L2_4b: MUX2 port map (d0 => '0', d1 => ng0, s => l2_term4_tmp, y => l2_term4);

    -- Суммируем всё (Цепочка ИЛИ)
    OR_L2_1: MUX2 port map (d0 => l2_term2, d1 => '1', s => l2_term1, y => l2_sum1);
    OR_L2_2: MUX2 port map (d0 => l2_term3, d1 => '1', s => l2_sum1, y => l2_sum2);
    OR_L2_3: MUX2 port map (d0 => l2_term4, d1 => '1', s => l2_sum2, y => led_out(2));

    ------------------------------------------------------------------------
    -- РЕАЛИЗАЦИЯ L1 = (G3*nG1) + (nG1*nG0) + (nG3*G1*G0)
    ------------------------------------------------------------------------

    -- Term1: G3 * nG1
    AND_L1_1: MUX2 port map (d0 => '0', d1 => ng1, s => g3, y => l1_term1);

    -- Term2: nG1 * nG0
    AND_L1_2: MUX2 port map (d0 => '0', d1 => ng0, s => ng1, y => l1_term2);

    -- Term3: nG3 * G1 * G0
    AND_L1_3a: MUX2 port map (d0 => '0', d1 => g1, s => ng3, y => l1_term3_part);
    AND_L1_3b: MUX2 port map (d0 => '0', d1 => g0, s => l1_term3_part, y => l1_term3);

    -- Суммируем
    OR_L1_1: MUX2 port map (d0 => l1_term2, d1 => '1', s => l1_term1, y => l1_sum1);
    OR_L1_2: MUX2 port map (d0 => l1_term3, d1 => '1', s => l1_sum1, y => led_out(1));

    ------------------------------------------------------------------------
    -- РЕАЛИЗАЦИЯ L0 = (nG3*nG0) + (G3*G0)
    ------------------------------------------------------------------------

    -- Term1: nG3 * nG0
    AND_L0_1: MUX2 port map (d0 => '0', d1 => ng0, s => ng3, y => l0_term1);

    -- Term2: G3 * G0
    AND_L0_2: MUX2 port map (d0 => '0', d1 => g0, s => g3, y => l0_term2);

    -- Суммируем
    OR_L0_Final: MUX2 port map (d0 => l0_term2, d1 => '1', s => l0_term1, y => led_out(0));

end structural;

-- Описание простых компонентов (нужно добавить в тот же файл или проект)

library IEEE;
use IEEE.STD_LOGIC_1164.ALL;

entity MUX2 is
    port ( d0, d1, s : in std_logic; y : out std_logic );
end MUX2;

architecture flow of MUX2 is
begin
    y <= d0 when s = '0' else d1;
end flow;

library IEEE;
use IEEE.STD_LOGIC_1164.ALL;

entity INV is
    port ( a : in std_logic; y : out std_logic );
end INV;

architecture flow of INV is
begin
    y <= not a;
end flow;
```

### Пояснение к коду для сдачи:

1.  **ДНФ:** Мы разбили каждый выход на сумму простых условий (минтермов). Например, $L_0$ горит, если (`nG3` И `nG0`) ИЛИ если (`G3` И `G0`).
2.  **Структурность:** В `architecture structural` нет операторов `and`, `or`, `not`. Есть только соединения (`port map`) компонентов `MUX2` и `INV`.
3.  **Логика на MUX:**
    - Когда вы видите `d0 => '0', d1 => X, s => Y` — это **И** (Y and X).
    - Когда вы видите `d0 => X, d1 => '1', s => Y` — это **ИЛИ** (Y or X).
