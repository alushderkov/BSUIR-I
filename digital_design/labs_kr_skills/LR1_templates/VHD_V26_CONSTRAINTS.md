Сначала объясню общие понятия, встречающиеся в XDC-файлах:

- **`set_property`**: Это команда в языке Tcl (который используется Vivado), чтобы задать определенное свойство для какого-либо объекта в вашем проекте.
- **`PACKAGE_PIN`**: Это свойство указывает на физический номер пина на корпусе микросхемы ПЛИС. У каждой ПЛИС (FPGA) на плате есть много-много выводов, и каждый из них имеет уникальное имя (например, P2, R2, U1). Выводы на плате Nexys4 DDR соединены с различными периферийными устройствами (светодиодами, переключателями, кнопками и т.д.).
- **`[get_ports {<имя_VHDL_порта>}]`**: Эта часть команды `set_property` используется для указания того, к какому порту в вашем VHDL-коде применяется свойство.
  - Для одиночного бита (например, `q_out`) это будет просто `{q_out}`.
  - Для векторов (например, `led_out[15]`) это будет `{led_out[15]}` для 15-го бита вектора `led_out`.
- **`IOSTANDARD`**: Это свойство определяет электрический стандарт (тип входных/выходных буферов) для данного пина. Различные стандарты имеют разные уровни напряжения, токи и временные характеристики.
- **`LVCMOS33`**: Расшифровывается как "Low Voltage CMOS 3.3V". Это очень распространенный стандарт ввода/вывода, который означает, что сигнал на этом пине будет работать с логическими уровнями, соответствующими 3.3 Вольтам (0 Вольт для логического '0', 3.3 Вольта для логического '1'). Большинство простых периферийных устройств на отладочных платах (светодиоды, переключатели) используют этот стандарт.
