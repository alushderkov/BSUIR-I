Структура компонента:Аппаратный модуль(Component)всегда состоит из интерфейса(Entity)и реализации(Architecture).Векторы и шины:Диапазоны задаются как(MSB downto LSB)для убывающей индексации или(LSB to MSB)для возрастающей.Оператор & используется для конкатенации векторов и битов.Атрибуты векторов:Позволяют писать масштабируемый код(например,в циклах и дженериках).Ключевые атрибуты:'high,'low,'left,'right,'length,'range,'reverse_range.Инициализация и заполнение(Aggregates):Присвоение всех нулей или единиц:(others=>'0').Позиционное присваивание:(7=>'1',3=>'1',others=>'0').Условные конструкции(Combinational):Потоковое присваивание:F<=A when(S='0')else B;(синтезируется в мультиплексор).Z-состояние:s2<=s0 when T='0'else'Z';(синтезируется в T-BUF).Порты(Port Mapping):Named(именованное):U0:COMP port map(A=>SigA,B=>SigB);— рекомендуется для читаемости,порядок не важен.Unnamed(позиционное):U0:COMP port map(SigA,SigB);— краткая запись,критичен порядок портов.Конструкции выбора:F<=A when S="00"else B when S="01"else C;— последовательная логика(приоритет).with S select F<=A when"00",B when"01",C when others;— параллельная логика(мультиплексор).Генерация структур:for...generate:Циклическое создание компонентов или логики.if...generate:Условное создание фрагментов схемы на этапе компиляции(например,выбор реализации в зависимости от generic флага).Конкатенация:Оператор&используется для объединения битов в векторы(например,s(i)&A(i+1)).Sequential If: VHDL if(condition)then -- statements elsif(condition)then -- statements else -- statements end if; Sequential Case: VHDL case selector is when value1=>-- statements when value2|value3=>-- statements when others=>-- mandatory for coverage end case; Process Template: VHDL label:process(sensitivity_list) variable var_name:type; begin -- sequential code end process label; Использование констант и типов:Для улучшения читаемости(например,состояния дешифратора)рекомендуется использовать constant или пользовательские типы.Арифметические операции:Использование библиотек ieee.numeric_std.all для работы с типами unsigned и signed(сложение+,вычитание-). Шаблон процесса для триггера(RTL): VHDL process(CLK,RST)--RST в списке,если он асинхронный begin if RST='1'then Q<='0'; elsif rising_edge(CLK)then Q<=D; end if; end process; Обратная связь:Для использования текущего значения выхода внутри процесса(например,в счетчиках или T-триггерах)необходимо использовать промежуточный сигнал(signal)или порт типа buffer.Массивы для памяти(Register Files):Память описывается через пользовательский тип двумерного массива. VHDL type t_reg_file is array(0 to M-1)of std_logic_vector(N-1 downto 0); signal REG_FILE:t_reg_file; Преобразование адреса:Для обращения к массиву по индексу,вектор std_logic_vector преобразуется в integer:to_integer(unsigned(Address)).(Требует ieee.numeric_std.all). Инициализация массивов:Внутри процесса по сигналу INIT массив инициализируется циклом for...loop.Паттерны сдвига(Concatenation):Реализуются через оператор&с использованием срезов векторов(slicing):Сдвиг влево(SIPO Left):nxt_state<=cur_state(1 to N-1)&SI;Сдвиг вправо(SIPO Right):nxt_state<=SI&cur_state(0 to N-2);Высокоимпедансное состояние выходов(Z-state):Применяется для мультиплексирования шин чтения(Output Enable):DO<=sreg when OE='1'else(others=>'Z');.Параметризация топологии(Generics):Структура полинома передается как generic параметр в виде битового вектора(std_logic_vector).'1' в$i$-й позиции означает наличие отвода,'0'— его отсутствие.Инициализация(Seed):M-sequence LFSR не может находиться в состоянии всех нулей(возникает зависание).Для старта требуется ненулевой вектор инициализации SEED(например,(0=>'1',others=>'0')).Атрибуты массивов:Для динамической генерации логики под полином произвольной длины применяются атрибуты'range,'reverse_range и'high.Генерация обратных связей(Combinational For-Loop):Использование цикла for...loop внутри процесса для вычисления следующего состояния(nxt_state)с использованием промежуточных переменных(мгновенное присваивание:=).
