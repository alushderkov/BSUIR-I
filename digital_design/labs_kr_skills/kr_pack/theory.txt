Уровни абстракции: Проектирование в VHDL делится на поведенческий (Behavioral), регистровых передач (RTL), логический (Logical) и технологический/схемотехнический (Netlist).Синтезируемость: Код делится на синтезируемый (преобразуемый в аппаратуру) и несинтезируемый (используемый для симуляции и отладки, например Time Description).Типы данных и логические состояния: Базовым типом является std_logic (стандарт IEEE 1164). Он имеет 9 состояний, где синтезируемыми обычно являются '0' (Forcing 0), '1' (Forcing 1), 'Z' (High impedance) и '-' (Don't care). Тип BIT поддерживает только '0' и '1'.Моделирование портов: Внешние порты (Ports) описывают интерфейс модуля (преобразуются в IBUF, OBUF, TBUF), а сигналы (Signals) — внутренние цепи.Ограничения портов:IN: Только чтение (нельзя присваивать значения внутри архитектуры).OUT: Только запись (нельзя читать значение порта внутри архитектуры).BUFFER: Выходной порт, допускающий как запись, так и чтение.INOUT: Двунаправленный порт, для корректной работы требует использования Z-состояний (Tri-state).Трехстабильные буферы (T-BUF): Используются для реализации двунаправленных шин (мультиплексирование на уровне шины). Состояние 'Z' (высокое сопротивление) отключает выход от общей линии.Структурное описание: Построение схемы через иерархию компонентов. Модуль состоит из объявления компонента (component declaration) и его вставки (component instantiation) в архитектуру.Параллельные (Concurrent) операторы: Все операторы в теле архитектуры выполняются одновременно и независимо от порядка их записи. К ним относятся:Простое присваивание сигналов.Условное присваивание (when-else).Селективное присваивание (with-select).Вставка компонентов.Операторы генерации (generate).Уровни синтеза:RTL-схема: Логическое представление проекта на уровне вентилей и мультиплексоров.Технологическая схема: Результат отображения на ресурсы конкретной FPGA (LUT, IBUF, OBUF).Масштабируемость: Использование generic позволяет создавать универсальные модули с настраиваемыми параметрами (разрядность, количество входов).Процесс (Process): Последовательный блок кода внутри параллельной архитектуры. Запускается при изменении любого сигнала из списка чувствительности (sensitivity list).Переменные (Variables) vs Сигналы (Signals):Variables: Объявляются только внутри процесса. Присваивание (:=) происходит мгновенно.Используются для промежуточных вычислений.Signals: Объявляются в архитектуре.Присваивание (<=) планируется и вступает в силу только после завершения процесса (в конце дельта-цикла).Оператор Wait: Используется для приостановки процесса. Процесс с wait не может иметь списка чувствительности.wait on [signals]: аналог списка чувствительности.wait until [condition]: срабатывание по условию (например, фронт клока).wait for [time]: задержка (только для симуляции).Моделирование задержек (Simulation only):Inertial Delay: Модель по умолчанию. Поглощает импульсы (spikes) короче времени задержки.Transport Delay: Модель линии передачи. Пропускает все импульсы без фильтрации.Reject: Настройка порога фильтрации импульсов в инерционной модели.Синтезируемость: Для синтеза комбинационной логики список чувствительности процесса должен содержать все сигналы, читаемые внутри. Отсутствие сигнала ведет к созданию нежелательных защелок (latches).Комбинационная логика: Реализация системы булевых функций без элементов памяти и обратных связей. Выход зависит только от текущего состояния входов.Синтез и Latches (Защелки):Если в процессе (process) не описано поведение для всех возможных состояний входов (отсутствует else в if или others в case), синтезатор создает Latch (защелку) для сохранения предыдущего значения.Для чистой комбинационной логики необходимо полностью определять выходы для всех комбинаций.Приоритетность:Конструкция if-elsif-else синтезирует цепочку приоритетной логики.Конструкция case (или with-select во внешней среде) синтезирует параллельную логику (мультиплексор), что обычно эффективнее по задержкам.Don’t Care ('-'): Использование знака прокола в case или сравнениях позволяет оптимизатору синтеза упрощать логику (минимизация по Картам Карно).Последовательностные схемы: Устройства, выход которых зависит не только от текущих входов, так и от предыдущих состояний (наличие памяти). Состоят из комбинационной логики и элементов памяти.Элементы памяти:Latches (Защелки): Управляются уровнем сигнала (level-sensitive).Flip-Flops (Триггеры): Управляются фронтом сигнала (edge-triggered).Синхронизация: Большинство последовательностных схем используют тактовый сигнал (CLK).rising_edge(CLK) — переход из '0' в '1'.falling_edge(CLK) — переход из '1' в '0'.Сброс (Reset/Clear):Асинхронный: Срабатывает мгновенно при изменении сигнала сброса, независимо от тактового сигнала (высший приоритет в процессе).Синхронный: Состояние изменяется только по фронту CLK.Ограничения синтеза: Технологические примитивы FPGA обычно содержат только D-триггеры. Другие типы (RS, T, JK) синтезируются на базе D-триггеров и комбинационной логики.Регистры (Registers): Узлы последовательностной логики, предназначенные для хранения и сдвига многоразрядных данных. Состоят из массива элементов памяти (защелок или триггеров).Типы регистров:Асинхронные (Latches): Запись данных (LD = '1') происходит по уровню (прозрачны). Синтезируются на базе D-защелок (LDCE).Синхронные (Flip-Flops): Изменение состояния строго привязано к фронту тактового сигнала CLK (и условию загрузки LD = '1'). Синтезируются на D-триггерах (FDRE, FDCE, FDPE).Регистровые файлы (Register Files): Быстродействующий массив регистров внутри процессора. Характеризуются количеством портов чтения (Read Ports) и записи (Write Ports). Типовая конфигурация для конвейера — 2 порта чтения и 1 порт записи (2R/1W), допускающая одновременное чтение двух операндов и запись результата.Сдвиговые регистры (Shift Registers): Выполняют последовательный сдвиг данных. Классифицируются по типу входа/выхода: SISO, SIPO, PISO, PIPO. Левый сдвиг (Shift Left) сдвигает от LSB к MSB, правый (Shift Right) — от MSB к LSB.Универсальные сдвиговые регистры (Universal): Объединяют режимы хранения, параллельной загрузки и сдвига в обе стороны через управляющий сигнал MODE.Сдвиговые регистры с линейной обратной связью (LFSR): Устройства, входной бит которых является линейной функцией (XOR) от предыдущего состояния регистра. Построены на базе арифметики полиномов в полях Галуа $GF(2)$.Характеристический полином $\phi(x)$: Определяет топологию обратных связей. Степень полинома $N = \deg(\phi(x))$ задает разрядность регистра.Примитивные неприводимые (Irreducible/Primitive) полиномы: Генерируют M-последовательности максимальной длины (орбиты). Период $L = 2^N - 1$.Приводимые (Reducible) полиномы: Генерируют последовательности с меньшим периодом (например, кольцевой счетчик).Топологии LFSR:Galois LFSR: Внутренние элементы XOR (расположены между триггерами). Обладает меньшей комбинационной задержкой (максимальная задержка — один XOR), оптимально для высокочастотного RTL-синтеза.Fibonacci LFSR: Внешние элементы XOR (собраны в каскад в цепи обратной связи). Задержка растет пропорционально числу отводов.Базовые паттерны (частные случаи):Ring Counter (Кольцевой счетчик): $\phi(x) = x^N + 1$. Обратная связь без логики. Генерирует One-hot / One-cold последовательности. Длина орбиты $L = N$.Johnson Counter (Счетчик Джонсона / Кольцевой счетчик Либиха): Обратная связь с инверсией $FB = \neg Q_{MSB}$. Длина орбиты $L = 2N$.SISR (Serial Input Signature Register):Сигнатурный анализатор. Осуществляет деление входного потока данных на характеристический полином с аппаратным вычислением остатка. Используется в криптографии, BIST (встроенном самотестировании) и для подсчета CRC.
