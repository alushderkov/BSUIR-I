Это **одноразрядная ячейка каскадного компаратора**. Она используется для сравнения двух многоразрядных двоичных чисел путём последовательного сравнения разрядов от старших к младшим. Такой подход позволяет получить результат сравнения после обработки всех разрядов.

### Зачем нужна такая ячейка?

Если нужно сравнить два n-разрядных числа, можно собрать цепочку из n таких ячеек. Каждая ячейка сравнивает один разряд (бит) и передаёт результат дальше. На вход самой старшей ячейки подаются начальные условия, а с выхода самой младшей снимается итоговый результат.

### Как она работает (логика)

У ячейки есть:

- **Входы**:
  - `a`, `b` – сравниваемые биты текущего разряда.
  - `gt_in`, `lt_in`, `eq_in` – результаты сравнения **более старших разрядов** (поступают с предыдущей ячейки).
    - `gt_in = '1'` – старшие разряды уже показали, что число A > B.
    - `lt_in = '1'` – старшие показали, что A < B.
    - `eq_in = '1'` – старшие разряды равны, решение ещё не принято.
- **Выходы**:
  - `gt_out`, `lt_out`, `eq_out` – обновлённые результаты с учётом текущего разряда (пойдут в следующую, более младшую ячейку).

**Правила формирования выходов:**

1. Если старшие разряды **уже определили отношение** (`gt_in=1` или `lt_in=1`), то текущий разряд ничего не меняет:
   - `gt_out = gt_in`, `lt_out = lt_in`, `eq_out = 0` (так как равенство уже нарушено).
2. Если старшие разряды **равны** (`eq_in=1`), то решение зависит от текущих битов:
   - Если `a > b` (т.е. `a=1` и `b=0`), то `gt_out=1`, `lt_out=0`, `eq_out=0`.
   - Если `a < b` (`a=0`, `b=1`), то `lt_out=1`, остальные нули.
   - Если `a = b` (`a=b=0` или `a=b=1`), то `eq_out=1`, `gt_out=lt_out=0`.

Математически это выражается формулами:

```
gt_out = gt_in  OR  (eq_in AND a AND NOT b)
lt_out = lt_in  OR  (eq_in AND NOT a AND b)
eq_out = eq_in AND (a XNOR b)   -- a XNOR b = (a AND b) OR (NOT a AND NOT b)
```

### Что делает этот VHDL-код?

- Он реализует эти формулы с помощью логических вентилей (`inv`, `and2`, `or2`).
- Вентили имеют параметризованные задержки (`tpd_and`, `tpd_or`, `tpd_inv`), что позволяет моделировать реальное время срабатывания схемы.
- Сигналы `gt_bit`, `lt_bit`, `eq_bit` – это результаты сравнения текущих битов без учёта старших разрядов:
  - `gt_bit = a AND NOT b` (A > B)
  - `lt_bit = NOT a AND b` (A < B)
  - `eq_bit = (a AND b) OR (NOT a AND NOT b)` (равенство)
- Затем они комбинируются с `eq_in` через дополнительные вентили, чтобы получить окончательные выходы.

### Пример использования

Допустим, мы строим 4-разрядный компаратор для чисел A(3..0) и B(3..0). Соединяем ячейки:

- Ячейка для старшего разряда (бит 3):
  - Входы `a=A(3)`, `b=B(3)`.
  - Начальные условия: `gt_in='0'`, `lt_in='0'`, `eq_in='1'` (предполагаем равенство, пока не доказано обратное).
  - Выходы идут в ячейку для бита 2.
- Ячейка для бита 2: берёт выходы предыдущей как свои входы, и т.д.
- После последнего (младшего) разряда выходы `gt_out`, `lt_out`, `eq_out` дают итог сравнения всего числа.

### Почему здесь используются `generic map`?

В коде внутри ячейки при подключении вентилей используется `generic map (tpd_inv)` и т.п. Это значит, что мы явно задаём время задержки для каждого вентиля, беря его из параметров самой ячейки. Если бы в объявлении компонента `inv` было значение по умолчанию (например, `generic (tpd : time := 1 ns);`), то можно было бы опустить `generic map`. Но здесь оно не указано, поэтому требуется явная передача.

При инстанцировании самой ячейки `comp_cell` вы **можете не писать `generic map`**, если вас устраивают значения по умолчанию (2 нс для AND/OR и 1 нс для INV), потому что в `entity` они уже заданы.

### Резюме

Это ячейка для построения многоразрядного компаратора методом последовательного сравнения. Она учитывает результат старших разрядов и формирует новый результат для младших. Такой подход широко используется в цифровой схемотехнике для создания компактных и быстрых компараторов.

Давайте разберём **внутреннюю работу одноразрядной ячейки** (`comp_cell`) на конкретном примере. Ячейка сравнивает два бита (`a`, `b`) и объединяет результат с уже известным отношением старших разрядов (`gt_in`, `lt_in`, `eq_in`). Выходы (`gt_out`, `lt_out`, `eq_out`) передаются дальше.

## 1. Какие сигналы формируются внутри?

### Этап 1: Сравнение текущих битов (без учёта старших)

- `gt_bit = a AND NOT b` – '1', если **текущий бит A больше B** (a=1, b=0).
- `lt_bit = NOT a AND b` – '1', если **текущий бит A меньше B** (a=0, b=1).
- `eq_bit = (a AND b) OR (NOT a AND NOT b)` – '1', если **текущие биты равны** (a=b).

Эти сигналы говорят о соотношении **только этого разряда**, игнорируя старшие.

### Этап 2: Учёт старших разрядов (каскадная логика)

- `gt_tmp = eq_in AND gt_bit` – если старшие разряды были равны (`eq_in=1`), то результат текущего бита `gt_bit` становится кандидатом на выходной `gt_out`.
- `lt_tmp = eq_in AND lt_bit` – аналогично для `lt_bit`.
- `gt_out = gt_in OR gt_tmp` – если старшие уже дали `gt_in=1`, то мы его сохраняем; иначе, если `eq_in=1` и текущий бит дал `gt_bit=1`, то `gt_out` становится 1.
- `lt_out = lt_in OR lt_tmp` – аналогично.
- `eq_out = eq_in AND eq_bit` – равенство сохраняется только если старшие были равны **и** текущие биты равны.

## 2. Почему такая логика работает?

Представим, что мы сравниваем два числа побитово, начиная со старшего разряда. Если в каком-то разряде обнаруживается неравенство, все младшие разряды уже не влияют на результат – они должны лишь повторить уже принятое решение. Если же старшие разряды равны, решение откладывается до первого неравного младшего разряда.

Именно это и делают формулы:

- Если `gt_in=1` (старшие уже показали A>B), то `gt_out=1` независимо от текущих битов, потому что `gt_tmp` может быть только 0 (так как `eq_in=0`), и `OR` с `gt_in` даёт 1. Аналогично для `lt_in`.
- Если `eq_in=1`, то решение зависит от текущих битов: `gt_out=gt_bit`, `lt_out=lt_bit`, `eq_out=eq_bit`. При равенстве битов `eq_bit=1`, и равенство сохраняется.

## 3. Пример для трёхразрядного числа

Возьмём числа:

- **A** = 1 0 1 (двоичное, старший бит = 1, средний = 0, младший = 1)
- **B** = 1 1 0 (старший = 1, средний = 1, младший = 0)

Проходим по разрядам сверху вниз.

### Разряд 2 (старший): a=1, b=1

- `gt_bit = 1 AND NOT 1 = 0`
- `lt_bit = NOT 1 AND 1 = 0`
- `eq_bit = (1 AND 1) OR (0 AND 0) = 1` (биты равны)

На вход ячейки для разряда 2 подаются начальные условия: `gt_in=0`, `lt_in=0`, `eq_in=1`.
Тогда:

- `gt_tmp = eq_in AND gt_bit = 1 AND 0 = 0`
- `lt_tmp = eq_in AND lt_bit = 1 AND 0 = 0`
- `gt_out = gt_in OR gt_tmp = 0 OR 0 = 0`
- `lt_out = lt_in OR lt_tmp = 0 OR 0 = 0`
- `eq_out = eq_in AND eq_bit = 1 AND 1 = 1`

После старшего разряда имеем промежуточный результат: `gt2=0`, `lt2=0`, `eq2=1`. То есть пока числа равны.

### Разряд 1 (средний): a=0, b=1

- `gt_bit = 0 AND NOT 1 = 0`
- `lt_bit = NOT 0 AND 1 = 1`
- `eq_bit = (0 AND 1) OR (1 AND 0) = 0`

Входные сигналы для этого разряда берутся с выхода предыдущего: `gt_in=gt2=0`, `lt_in=lt2=0`, `eq_in=eq2=1`.
Вычисляем:

- `gt_tmp = 1 AND 0 = 0`
- `lt_tmp = 1 AND 1 = 1`
- `gt_out = 0 OR 0 = 0`
- `lt_out = 0 OR 1 = 1`
- `eq_out = 1 AND 0 = 0`

После среднего разряда: `gt1=0`, `lt1=1`, `eq1=0`. Это означает: поскольку старшие были равны, а в текущем разряде A<B, то итоговое отношение – A<B. Дальнейшие разряды уже не изменят его.

### Разряд 0 (младший): a=1, b=0

- `gt_bit = 1 AND NOT 0 = 1`
- `lt_bit = NOT 1 AND 0 = 0`
- `eq_bit = (1 AND 0) OR (0 AND 1) = 0`

Входы: `gt_in=gt1=0`, `lt_in=lt1=1`, `eq_in=eq1=0`.

- `gt_tmp = 0 AND 1 = 0`
- `lt_tmp = 0 AND 0 = 0`
- `gt_out = 0 OR 0 = 0`
- `lt_out = 1 OR 0 = 1`
- `eq_out = 0 AND 0 = 0`

Выходы: `gt=0`, `lt=1`, `eq=0` – итоговый результат **A < B**, что верно (101 < 110).

## 4. Что происходит, если старшие уже определили отношение?

Возьмём другой пример: A=110, B=101 (старший разряд: 1 и 1 – равны, средний: 1 и 0 – A>B). После среднего разряда получим `gt1=1`, `lt1=0`, `eq1=0`. На младший разряд (a=0, b=1) придут `gt_in=1`, `lt_in=0`, `eq_in=0`. Тогда:

- `gt_tmp = 0 AND (gt_bit) = 0` (независимо от `gt_bit`)
- `lt_tmp = 0 AND (lt_bit) = 0`
- `gt_out = 1 OR 0 = 1`
- `lt_out = 0 OR 0 = 0`
- `eq_out = 0 AND eq_bit = 0`

Как видно, младший разряд просто повторяет решение старших (`gt_out=gt_in`), игнорируя свои биты. Это и есть каскадная логика.

## 5. Резюме

Каждая ячейка выполняет два действия параллельно:

1. Вычисляет отношение своих бит (`gt_bit`, `lt_bit`, `eq_bit`).
2. Комбинирует его с результатом от старших разрядов так, чтобы приоритет оставался за старшими.

Это достигается схемой из вентилей, где сигнал `eq_in` разрешает или блокирует влияние текущих бит.

Рассмотрим работу одноразрядной ячейки `comp_cell` построчно, сопоставляя логические выражения с конкретными вентилями в коде. Это поможет понять, как именно реализовано сравнение.

## Структура ячейки (архитектура `Structural`)

```vhdl
architecture Structural of comp_cell is
    component inv is generic (tpd : time); port (a : in STD_LOGIC; y : out STD_LOGIC); end component;
    component and2 is generic (tpd : time); port (a,b : in STD_LOGIC; y : out STD_LOGIC); end component;
    component or2 is generic (tpd : time); port (a,b : in STD_LOGIC; y : out STD_LOGIC); end component;

    signal not_a, not_b : STD_LOGIC;
    signal gt_bit, lt_bit, eq_bit : STD_LOGIC;
    signal a_and_b, not_a_and_not_b : STD_LOGIC;
    signal gt_tmp, lt_tmp : STD_LOGIC;
begin
    -- инверторы
    inv_a: inv generic map (tpd_inv) port map (a, not_a);
    inv_b: inv generic map (tpd_inv) port map (b, not_b);

    -- сравнение текущих битов
    and_gt: and2 generic map (tpd_and) port map (a, not_b, gt_bit);
    and_lt: and2 generic map (tpd_and) port map (not_a, b, lt_bit);
    and_eq1: and2 generic map (tpd_and) port map (a, b, a_and_b);
    and_eq2: and2 generic map (tpd_and) port map (not_a, not_b, not_a_and_not_b);
    or_eq:   or2  generic map (tpd_or)  port map (a_and_b, not_a_and_not_b, eq_bit); -- a xnor b

    -- каскадная логика
    and_gt2: and2 generic map (tpd_and) port map (eq_in, gt_bit, gt_tmp);
    or_gt:   or2  generic map (tpd_or)  port map (gt_in, gt_tmp, gt_out);

    and_lt2: and2 generic map (tpd_and) port map (eq_in, lt_bit, lt_tmp);
    or_lt:   or2  generic map (tpd_or)  port map (lt_in, lt_tmp, lt_out);

    and_eq_out: and2 generic map (tpd_and) port map (eq_in, eq_bit, eq_out);
end Structural;
```

## Пошаговое объяснение на примере

Возьмём конкретные значения:

- `a = '1'`, `b = '0'` (текущий разряд)
- Входные сигналы от старших разрядов: `gt_in = '0'`, `lt_in = '0'`, `eq_in = '1'` (старшие равны)

### 1. Инверторы

- `inv_a` получает `a = '1'`, выдаёт `not_a = '0'`.
- `inv_b` получает `b = '0'`, выдаёт `not_b = '1'`.
  _(Строки `inv_a` и `inv_b`)_

### 2. Вычисление отношения текущих битов

- `and_gt` (AND): `a` и `not_b` = `'1' and '1'` = `'1'` → `gt_bit = '1'`.
- `and_lt` (AND): `not_a` и `b` = `'0' and '0'` = `'0'` → `lt_bit = '0'`.
- `and_eq1` (AND): `a` и `b` = `'1' and '0'` = `'0'` → `a_and_b = '0'`.
- `and_eq2` (AND): `not_a` и `not_b` = `'0' and '1'` = `'0'` → `not_a_and_not_b = '0'`.
- `or_eq` (OR): `a_and_b` или `not_a_and_not_b` = `'0' or '0'` = `'0'` → `eq_bit = '0'`.

Итак, текущий бит даёт: `gt_bit=1` (A > B), `lt_bit=0`, `eq_bit=0` (не равны).

### 3. Каскадная логика (учёт старших)

- `and_gt2` (AND): `eq_in` и `gt_bit` = `'1' and '1'` = `'1'` → `gt_tmp = '1'`.
- `or_gt` (OR): `gt_in` или `gt_tmp` = `'0' or '1'` = `'1'` → `gt_out = '1'`.

- `and_lt2` (AND): `eq_in` и `lt_bit` = `'1' and '0'` = `'0'` → `lt_tmp = '0'`.
- `or_lt` (OR): `lt_in` или `lt_tmp` = `'0' or '0'` = `'0'` → `lt_out = '0'`.

- `and_eq_out` (AND): `eq_in` и `eq_bit` = `'1' and '0'` = `'0'` → `eq_out = '0'`.

Результат ячейки: `gt_out=1`, `lt_out=0`, `eq_out=0` – что означает: после учёта старших (которые были равны) и текущего разряда (A>B) окончательное отношение – A>B.

## Почему эта логика работает для каскада?

- Если `eq_in = '1'` (старшие равны), то выходные сигналы определяются текущими битами:  
  `gt_out = gt_bit`, `lt_out = lt_bit`, `eq_out = eq_bit`.  
  Это видно из формул: `gt_tmp = gt_bit`, `lt_tmp = lt_bit`, и `gt_in=lt_in=0`, поэтому `gt_out = gt_bit` и т.д.

- Если `eq_in = '0'` (старшие уже определили неравенство), то `gt_tmp = '0'`, `lt_tmp = '0'`, и тогда `gt_out = gt_in`, `lt_out = lt_in`, `eq_out = '0'`. То есть ячейка просто повторяет результат старших, игнорируя свои биты.

Таким образом, каждая ячейка либо **устанавливает** новое отношение (если старшие равны), либо **транслирует** уже принятое решение (если старшие неравны). Это и есть принцип каскадного компаратора.

## Пример с трёхразрядным компаратором

В `comp_3bit` три ячейки соединены последовательно. Для чисел A=101 (5) и B=110 (6) получается:

- **Бит 2** (a2=1, b2=1): eq_in=1 → выходы: gt2=0, lt2=0, eq2=1.
- **Бит 1** (a1=0, b1=1): eq_in=1 (от предыдущей) → вычисляем: gt_bit=0, lt_bit=1, eq_bit=0 → gt1=0, lt1=1, eq1=0.
- **Бит 0** (a0=1, b0=0): eq_in=0 (от бита 1) → независимо от a0,b0 выходы: gt0=gt1=0, lt0=lt1=1, eq0=0.

Итог: gt=0, lt=1, eq=0 – верно (5 < 6).

Все эти вычисления выполняются параллельно, с учётом задержек каждого вентиля, что и создаёт временные диаграммы с переходами сигналов.

Давайте разберём архитектуру `Structural` одноразрядной ячейки `comp_cell` построчно, чтобы понять, зачем нужен каждый сигнал и каждый вентиль. Мы увидим, как из простых логических элементов строится каскадный компаратор.

## Общая идея

Ячейка сравнивает два бита `a` и `b`, но учитывает результат сравнения **более старших разрядов**, который приходит на входы `gt_in`, `lt_in`, `eq_in`:

- `gt_in = '1'` – старшие разряды уже показали, что число A > B.
- `lt_in = '1'` – старшие показали, что A < B.
- `eq_in = '1'` – старшие разряды равны, решение ещё не принято.

На выходе ячейка выдаёт `gt_out`, `lt_out`, `eq_out` – обновлённые результаты, которые пойдут в следующий (младший) разряд.

**Правила работы:**

1. Если старшие разряды **уже определили** отношение (`gt_in=1` или `lt_in=1`), то текущий разряд ничего не меняет: просто передаём `gt_in` и `lt_in` дальше, а `eq_out` делаем `0`.
2. Если старшие разряды **равны** (`eq_in=1`), то решение зависит от текущих битов:
   - Если `a > b` → `gt_out=1`, остальные нули.
   - Если `a < b` → `lt_out=1`, остальные нули.
   - Если `a = b` → `eq_out=1`, остальные нули.

## Построчная расшифровка кода

### 1. Объявление компонентов и внутренних сигналов

```vhdl
component inv is generic (tpd : time); port (a : in STD_LOGIC; y : out STD_LOGIC); end component;
component and2 is generic (tpd : time); port (a,b : in STD_LOGIC; y : out STD_LOGIC); end component;
component or2 is generic (tpd : time); port (a,b : in STD_LOGIC; y : out STD_LOGIC); end component;
```

Это заготовки для базовых логических элементов: инвертора, двухвходового И и двухвходового ИЛИ. Они будут использованы для построения схемы. Задержки (`tpd`) позволяют моделировать реальное время срабатывания.

```vhdl
signal not_a, not_b : STD_LOGIC;
signal gt_bit, lt_bit, eq_bit : STD_LOGIC;
signal a_and_b, not_a_and_not_b : STD_LOGIC;
signal gt_tmp, lt_tmp : STD_LOGIC;
```

- `not_a`, `not_b` – инвертированные значения входов.
- `gt_bit`, `lt_bit`, `eq_bit` – **результаты сравнения только текущих битов** (без учёта старших):
  - `gt_bit = a AND NOT b` (A > B)
  - `lt_bit = NOT a AND b` (A < B)
  - `eq_bit = (a AND b) OR (NOT a AND NOT b)` (равенство)
- `a_and_b` и `not_a_and_not_b` – промежуточные сигналы для вычисления `eq_bit` через ИЛИ.
- `gt_tmp`, `lt_tmp` – вспомогательные сигналы, показывающие, даёт ли текущий бит вклад в выходы **при условии, что старшие равны**.

### 2. Инверторы

```vhdl
inv_a: inv generic map (tpd_inv) port map (a, not_a);
inv_b: inv generic map (tpd_inv) port map (b, not_b);
```

Создаём инвертированные копии входных битов. Они понадобятся для всех дальнейших вычислений (например, для `lt_bit` нужно `not_a`, для `eq_bit` – `not_a` и `not_b`).

### 3. Сравнение текущих битов

```vhdl
and_gt: and2 generic map (tpd_and) port map (a, not_b, gt_bit);
```

Это вентиль И: на выходе `gt_bit = a AND not_b`. Если `a=1` и `b=0`, то `gt_bit=1` – сигнал, что текущий бит A больше B.

```vhdl
and_lt: and2 generic map (tpd_and) port map (not_a, b, lt_bit);
```

`lt_bit = not_a AND b`. Если `a=0` и `b=1`, то `lt_bit=1` – сигнал, что текущий бит A меньше B.

```vhdl
and_eq1: and2 generic map (tpd_and) port map (a, b, a_and_b);
and_eq2: and2 generic map (tpd_and) port map (not_a, not_b, not_a_and_not_b);
or_eq:   or2  generic map (tpd_or)  port map (a_and_b, not_a_and_not_b, eq_bit);
```

Эти три вентиля вычисляют равенство битов:

- `a_and_b = a AND b` – истинно, если оба бита равны 1.
- `not_a_and_not_b = not_a AND not_b` – истинно, если оба бита равны 0.
- `eq_bit = a_and_b OR not_a_and_not_b` – истинно, когда биты равны (оба 0 или оба 1). Это эквивалентно `a XNOR b`.

**Итог этапа 3:** мы получили три сигнала, описывающих отношение **только этого разряда**:

- `gt_bit` – A > B в текущем разряде,
- `lt_bit` – A < B в текущем разряде,
- `eq_bit` – A = B в текущем разряде.

### 4. Каскадная логика (учёт старших разрядов)

Теперь нужно объединить информацию от старших разрядов (`gt_in`, `lt_in`, `eq_in`) с результатами текущего разряда.

```vhdl
and_gt2: and2 generic map (tpd_and) port map (eq_in, gt_bit, gt_tmp);
```

`gt_tmp = eq_in AND gt_bit`. Если старшие равны (`eq_in=1`), то `gt_tmp` повторяет `gt_bit`; иначе `gt_tmp=0`.

```vhdl
or_gt:   or2  generic map (tpd_or)  port map (gt_in, gt_tmp, gt_out);
```

`gt_out = gt_in OR gt_tmp`. Если старшие уже дали `gt_in=1`, то выход сразу 1. Иначе, если старшие равны и текущий бит дал `gt_bit=1`, то `gt_tmp=1` и выход становится 1.

```vhdl
and_lt2: and2 generic map (tpd_and) port map (eq_in, lt_bit, lt_tmp);
or_lt:   or2  generic map (tpd_or)  port map (lt_in, lt_tmp, lt_out);
```

Аналогично для `lt_out`.

```vhdl
and_eq_out: and2 generic map (tpd_and) port map (eq_in, eq_bit, eq_out);
```

`eq_out = eq_in AND eq_bit`. Если старшие равны (`eq_in=1`) и текущие биты равны (`eq_bit=1`), то равенство сохраняется (`eq_out=1`). В противном случае `eq_out=0`.

## Как это работает в каскаде?

Рассмотрим два случая:

**Случай 1: старшие уже решили (eq_in=0)**

- `gt_tmp = 0`, `lt_tmp = 0`, поэтому `gt_out = gt_in`, `lt_out = lt_in`, `eq_out = 0`. Ячейка просто повторяет решение старших, игнорируя свои биты.

**Случай 2: старшие равны (eq_in=1)**

- `gt_tmp = gt_bit`, `lt_tmp = lt_bit`, `eq_out = eq_bit`. При этом `gt_in` и `lt_in` в этом случае должны быть 0 (так как старшие равны), поэтому `gt_out = gt_bit`, `lt_out = lt_bit`. Ячейка выдаёт результат сравнения своих битов.

Таким образом, каждая ячейка либо транслирует уже принятое решение, либо принимает новое решение на основе своих битов. Соединяя такие ячейки последовательно от старшего разряда к младшему, мы получаем полноценный многоразрядный компаратор.

## Визуализация работы на примере

Пусть сравниваем числа A = 101, B = 110 (старший бит – самый левый). Идём от старшего к младшему:

- **Ячейка для бита 2 (a=1, b=1)**:  
  `gt_bit=0`, `lt_bit=0`, `eq_bit=1`.  
  Входы от ещё более старших (их нет) заданы как `gt_in=0, lt_in=0, eq_in=1`.  
  Выходы: `gt2=0, lt2=0, eq2=1` (равенство пока).

- **Ячейка для бита 1 (a=0, b=1)**:  
  `gt_bit=0`, `lt_bit=1`, `eq_bit=0`.  
  Входы: `gt_in=gt2=0, lt_in=lt2=0, eq_in=eq2=1`.  
  Тогда `gt_tmp = 1 AND 0 =0`, `lt_tmp=1 AND 1=1`, `gt_out=0 OR 0=0`, `lt_out=0 OR 1=1`, `eq_out=1 AND 0=0`.  
  Получаем `gt1=0, lt1=1, eq1=0` (уже ясно, что A<B).

- **Ячейка для бита 0 (a=1, b=0)**:  
  `gt_bit=1`, `lt_bit=0`, `eq_bit=0`.  
  Входы: `gt_in=gt1=0, lt_in=lt1=1, eq_in=eq1=0`.  
  Так как `eq_in=0`, то `gt_tmp=0`, `lt_tmp=0`, `gt_out=gt_in=0`, `lt_out=lt_in=1`, `eq_out=0`.  
  Итог: `gt=0, lt=1, eq=0` – правильно, 101 < 110.

## Резюме

Каждый сигнал и вентиль выполняют строго определённую роль:

- Инверторы готовят инверсные значения.
- Первая группа вентилей вычисляет отношение текущих битов.
- Вторая группа комбинирует это с решением от старших разрядов, реализуя каскадную логику.

Благодаря такой структуре компаратор можно легко расширить на любое количество разрядов, просто добавив ещё одну ячейку.

Эти строки реализуют **каскадную логику** – они учитывают результат сравнения старших разрядов и объединяют его с результатом сравнения текущих битов. Давайте разберем каждую строку подробно.

### Исходные сигналы:

- `eq_in` – приходит из старшей ячейки: `1` означает, что все предыдущие (более старшие) разряды равны.
- `gt_in`, `lt_in` – результаты старших разрядов: если один из них `1`, то общее отношение уже определено (A>B или A<B).
- `gt_bit`, `lt_bit`, `eq_bit` – результаты сравнения **только текущих битов** a и b (вычислены ранее).

### Логика для `gt_out` (выходной сигнал "больше")

```vhdl
and_gt2: entity work.and2_d Generic map (tpd_and) Port map (eq_in, gt_bit, gt_tmp);
or_gt:   entity work.or2_d  Generic map (tpd_or)  Port map (gt_in, gt_tmp, gt_out);
```

- **and_gt2** – это вентиль И. Он вычисляет:  
  `gt_tmp = eq_in AND gt_bit`.  
  Если старшие разряды равны (`eq_in = 1`), то `gt_tmp` равен `gt_bit` (результату сравнения текущих битов). Если старшие не равны (`eq_in = 0`), то `gt_tmp = 0` независимо от `gt_bit`.
- **or_gt** – вентиль ИЛИ. Он даёт:  
  `gt_out = gt_in OR gt_tmp`.
  - Если старшие уже дали `gt_in = 1`, то выход сразу `1` (через ИЛИ).
  - Если `gt_in = 0`, то выход равен `gt_tmp`.

**Итог для `gt_out`:**

- Если старшие решили, что A > B (`gt_in = 1`), то `gt_out = 1` (игнорируем текущий разряд).
- Если старшие равны (`eq_in = 1`), то `gt_out = gt_bit` (результат текущего сравнения).
- Если старшие решили, что A < B (`lt_in = 1`, `gt_in = 0`), то `gt_out = 0` (так как `gt_tmp = 0`).

### Логика для `lt_out` (выход "меньше")

```vhdl
and_lt2: entity work.and2_d Generic map (tpd_and) Port map (eq_in, lt_bit, lt_tmp);
or_lt:   entity work.or2_d  Generic map (tpd_or)  Port map (lt_in, lt_tmp, lt_out);
```

Работает симметрично:

- `lt_tmp = eq_in AND lt_bit`.
- `lt_out = lt_in OR lt_tmp`.
- Если старшие уже дали `lt_in = 1`, то `lt_out = 1`.
- Если старшие равны, то `lt_out = lt_bit`.
- Если старшие дали `gt_in = 1`, то `lt_out = 0`.

### Логика для `eq_out` (выход "равно")

```vhdl
and_eq_out: entity work.and2_d Generic map (tpd_and) Port map (eq_in, eq_bit, eq_out);
```

Это просто И: `eq_out = eq_in AND eq_bit`.

- Если старшие равны (`eq_in = 1`) **и** текущие биты равны (`eq_bit = 1`), то равенство сохраняется (`eq_out = 1`).
- Во всех остальных случаях `eq_out = 0` (старшие уже неравны, либо текущие биты неравны).

### Зачем это нужно?

Эти вентили реализуют **приоритет старших разрядов**. Если старшие разряды уже определили отношение (A>B или A<B), текущий разряд не может его изменить – он просто передаёт это решение дальше. Если же старшие равны, то текущий разряд сам определяет результат.

### Пример работы (для закрепления)

Пусть старшие разряды уже дали `gt_in = 1`, `lt_in = 0`, `eq_in = 0`. Тогда:

- `gt_tmp = 0 AND gt_bit = 0` → `gt_out = 1 OR 0 = 1`
- `lt_tmp = 0 AND lt_bit = 0` → `lt_out = 0 OR 0 = 0`
- `eq_out = 0 AND eq_bit = 0`

Выходы `(1,0,0)` – повторяют `gt_in`, игнорируя текущие биты. Всё верно.

Если старшие равны (`eq_in = 1`), например `gt_in=0, lt_in=0`, а текущие биты: `a=1, b=0` → `gt_bit=1, lt_bit=0, eq_bit=0`. Тогда:

- `gt_tmp = 1 AND 1 = 1` → `gt_out = 0 OR 1 = 1`
- `lt_tmp = 1 AND 0 = 0` → `lt_out = 0 OR 0 = 0`
- `eq_out = 1 AND 0 = 0`

Получаем `(1,0,0)` – правильно, так как при равных старших и A>B в текущем разряде итог A>B.

Таким образом, каждая пара вентилей (И + ИЛИ) реализует функцию:

- Если `eq_in=1`, то пропустить сигнал сравнения текущего бита (`gt_bit` или `lt_bit`) на выход.
- Если `eq_in=0`, то заблокировать его и использовать только `gt_in`/`lt_in`.

А для равенства просто проверяется, что и старшие равны, и текущие равны.
